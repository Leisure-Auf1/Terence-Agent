# 模块化设计说明 — 实验 2

## 模块关系图

```
┌──────────────────────────────────────────────────────────────┐
│                         main.py                              │
│                   命令行交互层 (CLI)                           │
│         解析 sys.argv → 分发到对应服务函数                        │
│         显示输出给用户                                           │
└──────────────────────┬───────────────────────────────────────┘
                       │ 调用
                       ▼
┌──────────────────────────────────────────────────────────────┐
│                     task_service.py                          │
│                   业务逻辑层 (Service)                         │
│         添加任务 / 查询列表 / 完成任务                           │
│         维护编号递增、状态转换验证                                │
│         不关心数据怎么存、不关心用户界面                           │
└──────────────────────┬───────────────────────────────────────┘
                       │ 调用
          ┌────────────┴────────────┐
          ▼                         ▼
┌─────────────────────┐  ┌─────────────────────────────────────┐
│   task_model.py     │  │      task_storage.py                │
│   数据模型 (Model)   │  │      存储层 (Storage)                │
│   Task 数据类        │  │  JSON 文件的读/写                    │
│   to_dict/from_dict │  │  处理文件不存在的情况                   │
│   mark_done/is_done │  │  封装文件路径和 I/O 细节               │
└─────────────────────┘  └─────────────────────────────────────┘
```

## 各模块职责

| 模块 | 职责 | 对外暴露 |
|------|------|---------|
| `main.py` | 命令行交互，解析用户输入，显示结果 | 无（入口文件） |
| `task_service.py` | 业务逻辑：添加、查询、完成任务 | `add()`, `list_all()`, `done()` |
| `task_model.py` | 任务数据结构定义 | `Task` 数据类 |
| `task_storage.py` | JSON 文件持久化 | `load_all()`, `save_all()` |

## 重构要点

1. **单一职责**：每个模块只做一件事。main.py 不碰文件 I/O，storage 不碰业务规则
2. **依赖方向**：`main → service → (model, storage)`，下层不依赖上层
3. **每个函数 ≤ 25 行**：最大函数为 21 行，均符合要求
4. **意图命名**：`add()`/`list_all()`/`done()`/`load_all()`/`save_all()`
5. **功能零回归**：所有原功能（add/list/done/空列表/错误提示）完整保留

## 层间数据流

```
用户输入 → main.py(解析) → task_service.add/起来/done()
            → task_model.Task(构造数据)
            → task_storage.save_all(持久化)
            → 结果返回 main.py(展示)
```
