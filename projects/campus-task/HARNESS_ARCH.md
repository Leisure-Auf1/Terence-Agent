## 各模块职责

| 模块 | 输入 | 输出 | 职责 |
|------|------|------|------|
| `prompt_builder` | 用户输入 + 任务状态 | 结构化 prompt | 将问题转化为 LLM 可理解的格式 |
| `mock_model` | prompt | JSON 字符串 | 模拟 LLM 理解并决策 |
| `parse_model_output` | 模型输出 JSON | 动作指令 dict | 将 JSON 解析为可执行指令 |
| `guardrail` | 动作指令 | 通过/拒绝+理由 | 安全护栏，过滤危险操作 |
| `execute_tool` | 动作指令 | 执行结果字符串 | 调用 CampusTask 真实函数 |
| `write_trace` | 事件 dict | 追加到 trace.jsonl | 全流程日志记录 |
| `run_eval` | 评测用例列表 | 准确率+详细信息 | 自动评估 AI 决策质量 |

## 数据流

```
输入 → prompt_builder → mock_model → parser → guardrail → execute_tool → 结果
                                                          ↕
                                                     trace.jsonl
```

## 安全设计

"删除所有任务"等危险操作被 `guardrail()` 拦截，永不执行。这体现了
**human-in-the-loop** 原则：当模型提出有副作用的工具调用时，系统暂停并
拒绝执行，等待人工手动操作。
