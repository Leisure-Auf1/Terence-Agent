# 📊 报告撰写工作流

> **用途**: 项目/技术/实验报告的快速撰写
> **工作量**: 比学术论文轻量，聚焦于清晰传达信息

## 流程

```
Phase 1: 需求确认 (Coordinator)
  ├─ 报告类型（项目报告 / 技术报告 / 实验报告）
  ├─ 目标读者（技术团队 / 管理层 / 客户）
  └─ 格式要求（DOCX / PDF / Markdown）
      │
Phase 2: 背景收集 (Researcher)
  ├─ 查找相关背景资料
  ├─ 收集已有数据和结果
  └─ 输出：背景笔记
      │
Phase 3: 撰写报告 (Writer)
  ├─ 按报告模板组织
  │   摘要 → 背景 → 方法 → 结果 → 讨论 → 结论
  └─ 每个部分基于已有数据
      │
Phase 4: 质量检查 (Reviewer)
  ├─ 内容完整性检查
  ├─ 数据一致性检查
  ├─ 格式检查
  └─ 提交审稿报告
      │
Phase 5: 修改 (Writer)
  └─ 根据反馈修改
      │
Phase 6: 交付 (Coordinator)
  └─ 最终报告文件
      │
Post-Task: 日志
```

## 报告质量检查清单

- [ ] 摘要包含所有关键信息
- [ ] 背景说明充分
- [ ] 方法描述可复现
- [ ] 结果有数据支持
- [ ] 讨论有意义
- [ ] 结论有行动建议
- [ ] 格式一致
- [ ] 所有引用/链接可访问
