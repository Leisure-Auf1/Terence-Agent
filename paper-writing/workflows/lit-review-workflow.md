# 📚 文献综述工作流

> **参考**: Feynman `/lit` + Hermes `research-paper-writing` Phase 1
> **用途**: 对研究课题撰写完整的文献综述

## 流程

```
Phase 1: 确定范围 (Coordinator)
  ├─ 明确综述的主题边界
  ├─ 确定关键词和搜索策略
  └─ 设定时间范围和相关标准
      │
Phase 2: 三篇迭代搜索 (Researcher)
  ├─ Round 1 (广度): 4-6并行搜索覆盖不同角度
  │   → 收集论文 + 提取关键概念
  ├─ Round 2 (深度): 追踪引用 + 新术语
  │   → 深入最相关的论文
  ├─ Round 3 (针对性): 填补空白
  │   → 缺失基线 + 最新成果
  └─ 停止: 一轮搜索 >80% 已收集 → 饱和
      │
Phase 3: 信息提取 (Researcher)
  ├─ 提取每篇论文的关键信息
  ├─ 识别子主题
  ├─ 分析共识与分歧
  └─ 发现研究空白
      │
Phase 4: 撰写综述 (Writer)
  ├─ 按文献综述模板组织
  ├─ 每个子主题独立章节
  ├─ 研究空白章节
  └─ 每条引用标记验证状态
      │
Phase 5: 引用验证 (Reviewer)
  ├─ 逐一验证所有引用
  └─ 提交审稿报告
      │
Phase 6: 迭代 (Coordinator/Writer)
  └─ 根据审稿反馈修改
      │
Phase 7: 交付
      │
Post-Task: 日志
```

## 引用验证（强制执行）

```
对每篇引用论文:
  Step 1: 搜索确认 → arXiv / Semantic Scholar
  Step 2: 内容验证 → 声明确实出现在原文
  Step 3: BibTeX → 通过 DOI 获取
  Step 4: 状态标记 → ✅ / ❌
```

## 文献综述质量检查清单

- [ ] 覆盖了所有相关子领域
- [ ] 关键论文没有遗漏
- [ ] 共识和分歧都已分析
- [ ] 研究空白明确
- [ ] 所有引用已验证
- [ ] 结构清晰、逻辑连贯
